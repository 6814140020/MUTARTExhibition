# Exhibition 2569 — ระบบจัดการงบประมาณ สต๊อก และจัดซื้อ

เว็บแอปสำหรับทดแทนไฟล์ Excel เดิม มีระบบล็อกอินจริง แบ่งสิทธิ์ผู้อนุมัติ/สมาชิก
ข้อมูล realtime หลายคนใช้พร้อมกันได้

**Stack**: React (Vite) + Tailwind + Supabase (Postgres, Auth, Realtime) + Vercel (hosting)

---

## 1. ตั้งค่า Supabase (ฐานข้อมูล + ระบบล็อกอิน)

1. ไปที่ https://supabase.com สมัคร/ล็อกอิน แล้วกด "New project" (เลือก region Singapore ใกล้ไทยสุด)
2. ตั้งรหัสผ่านฐานข้อมูล เก็บไว้ให้ดี (ไม่ใช่รหัสที่ใช้ล็อกอินแอป)
3. รอโปรเจกต์สร้างเสร็จ (~2 นาที) แล้วเข้าไปที่เมนู **SQL Editor** ทางซ้าย
4. เปิดไฟล์ `supabase/schema.sql` ในโปรเจกต์นี้ copy โค้ดทั้งหมดไปวางใน SQL Editor แล้วกด **Run**
   - ไฟล์นี้จะสร้างตารางทั้งหมด (ทีมงาน, งบ, ธุรกรรม, สต๊อก, จัดซื้อ) พร้อมกำหนดสิทธิ์การเข้าถึง (RLS) และเปิด Realtime ให้แล้ว
5. ไปที่เมนู **Authentication > Providers** เช็คว่า Email เปิดใช้งานอยู่ (เปิดเป็นค่าเริ่มต้นอยู่แล้ว)
   - ถ้าไม่อยากให้ทุกคนต้องยืนยันอีเมลก่อนใช้งาน ไปที่ **Authentication > Settings** ปิด "Confirm email"
6. ไปที่เมนู **Project Settings > API** จะเห็น
   - `Project URL` → เอาไปใส่ `VITE_SUPABASE_URL`
   - `anon public` key → เอาไปใส่ `VITE_SUPABASE_ANON_KEY`

---

## 2. รันโปรเจกต์บนเครื่องตัวเอง (ทดสอบก่อน deploy)

```bash
# ติดตั้ง Node.js 18+ ก่อน ถ้ายังไม่มี

npm install
cp .env.example .env.local
# แก้ .env.local ใส่ค่า VITE_SUPABASE_URL และ VITE_SUPABASE_ANON_KEY ที่ได้จากขั้นตอนที่ 1

npm run dev
# เปิด http://localhost:5173
```

ทดสอบ:
1. กด "สมัครสมาชิก" สร้างบัญชีแรกของตัวเอง
2. ไปที่ Supabase Dashboard → Table Editor → ตาราง `profiles` → แก้ `role` ของบัญชีตัวเองเป็น `admin`
   (คนแรกต้องตั้งเองผ่าน dashboard เพราะระบบยังไม่มีใครเป็น admin ให้กดอนุมัติ)
3. ล็อกอินใหม่ จะเห็นสิทธิ์ "ผู้อนุมัติ" ที่แถบด้านซ้าย

---

## 3. Deploy ขึ้นใช้งานจริงด้วย Vercel

1. Push โปรเจกต์นี้ขึ้น GitHub (สร้าง repo ใหม่แล้ว push โค้ดทั้งหมด)
2. ไปที่ https://vercel.com เชื่อมบัญชี GitHub แล้วกด **New Project** เลือก repo นี้
3. ตอนตั้งค่า ใส่ Environment Variables:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
4. กด Deploy รอสักครู่ จะได้ลิงก์ เช่น `https://exhibition-2569.vercel.app` ใช้งานได้จริงทันที ทุกคนเปิดผ่านมือถือได้

---

## 4. การใช้งานระบบสิทธิ์

- **สมาชิก (member)**: ยื่นขอเบิกเงิน/ของ, บันทึกการซื้อของ, ดูภาพรวมทั้งหมด แต่กดอนุมัติเองไม่ได้
- **ผู้อนุมัติ (admin)**: ทำทุกอย่างที่สมาชิกทำได้ + กดอนุมัติ/ไม่อนุมัติทุกรายการ + จัดการทีมงานและสต๊อกเริ่มต้น
- การตั้งใครเป็น admin ทำผ่าน Supabase Table Editor → ตาราง `profiles` → แก้ค่า `role`
  (หรือให้ admin คนแรกไปตั้งให้คนอื่นในหน้า "ทีมงาน" ของแอปได้เลย)

---

## 5. โครงสร้างระบบ

- `funds` — 3 ก้อนเงิน (งบหลัก/งบสำรอง/ค่าน้ำมัน)
- `transactions` — บันทึกรายรับ-รายจ่ายผูกกับก้อนเงิน มี workflow อนุมัติ
- `stock_items` + `stock_logs` — สต๊อกของ และประวัติการเบิก (หักสต๊อกอัตโนมัติเมื่ออนุมัติ)
- `purchases` — บันทึกการซื้อของ ผูกกับก้อนเงินที่หัก
- `profiles` — ข้อมูลทีมงาน + สิทธิ์ (role)
- แดชบอร์ดคำนวณยอดคงเหลือ/กราฟจากรายการที่สถานะ "ผ่าน" เท่านั้น

## 6. สิ่งที่ยังต่อยอดได้

- อัปโหลดไฟล์หลักฐานจริง (ใช้ Supabase Storage แทนแปะลิงก์ Drive)
- แจ้งเตือนผ่าน Line Notify เมื่อมีรายการรออนุมัติ หรือของใกล้หมด
- Export รายงานเป็น Excel/PDF จากหน้าแดชบอร์ด
- ประวัติการแก้ไข (audit log) แบบละเอียดกว่านี้
